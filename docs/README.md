# Documentation Structure

This directory contains all project documentation, organized into logical folders for better maintainability and clarity.

## Folder Structure

- **api/**: API specifications, backend architecture, and API setup guides
  - OpenAPI specifications
  - Backend skeleton documentation
  - Meta API setup instructions
  - OAuth implementation details

- **architecture/**: Application structure and architectural documentation
  - Overall app structure diagrams and descriptions

- **database/**: Database schemas, migrations, and automation examples
  - Database schema files
  - SQL automation scripts

- **operations/**: Operational guides, environment setup, and maintenance
  - Operations guide
  - Environment variables documentation
  - Auto-update procedures

- **planning/**: Project planning, milestones, phases, and implementation planning
  - Phase completion reports
  - Milestone plans and testing
  - Implementation checklists and backlogs

- **releases/**: Version history and release documentation
  - Version release notes and changelogs

- **testing/**: Testing documentation and results
  - Milestone testing reports
  - Test results

- **webhooks/**: Webhook samples and testing documentation
  - Webhook sample payloads
  - Webhook testing procedures

## Guidelines

- Place new documentation files in the appropriate folder based on their purpose
- For feature-specific documentation, consider placing it in a `documentation/` subfolder within the relevant service or module directory
- Maintain consistent naming conventions for files
- Update this README when adding new folders or changing the structure
